# 자소서 채점 기준 명세서 (생산/제조 직무 MVP)

> **버전**: v0.1 (초안)
> **대상 직무**: 생산/제조 (생산기술·품질·공정기술)
> **목적**: 합격 자소서 분포 비교 기반 AI 진단 → 멘토 상담 연결
> **상태**: 설계 합의 완료, 코드 구현 전 명세

---

## 0. 설계 철학

이 모델은 단순한 "자소서 절대평가"가 아니다.
**"합격 자소서와 얼마나 닮았는가"** 를 측정하는 비교 모델이다.

### 3대 원칙

1. **감점 금지, 가점 중심**
   - 항목이 없다고 점수를 깎지 않는다.
   - 있으면 더하고(보너스), 애매하면 멘토로 보낸다.

2. **합격자 분포가 기준선**
   - 이론적 "좋은 자소서"가 아니라, 실제 합격 자소서 50개+의 통계가 기준.
   - 사용자에게는 항상 "합격자 평균 대비 당신의 위치"로 표현.

3. **정직함 = 매출**
   - AI가 판단하기 애매한 구간은 솔직하게 "신뢰도 낮음"으로 표시.
   - 그 애매구간이 곧 유료 멘토 상담으로 가는 자연스러운 전환 지점.

## 1. 보안 원칙 (절대 준수)

합격 자소서는 사업의 핵심 자산. 원문은 어디에도 노출되지 않는다.

```
합격 자소서 원문
   ↓ [오프라인 1회만] 임베딩 변환
벡터 + 통계치만 저장  ← 앱이 사용하는 유일한 데이터
   ↓
원문은 앱/repo/사용자화면 어디에도 존재하지 않음
```

- ❌ 합격 자소서 원문을 사용자 화면에 출력 금지
- ❌ "이 합격자는 이렇게 썼어요" 식 직접 인용 금지
- ❌ GitHub repo(public/private 무관)에 원문 저장 금지 — 벡터 파일만
- ✅ 사용자는 "합격자 평균 대비 나의 위치" 숫자/비교만 확인
- ✅ 임베딩 벡터는 원문 역산이 사실상 불가능 (보안의 핵심 근거)

## 2. 직무 범위

```
생산/제조 (벡터 DB는 통합 — 합격 자소서 결이 유사함)
├── 생산기술 (생기)    설비/자동화/라인 설계
├── 품질               불량/검사/개선활동
└── 공정기술           공정설계/표준화/IE
```

- **벡터 DB**: 세 직무 통합 (데이터 50개를 쪼개지 않아 통계적으로 유리)
- **JD 키워드**: 직무별로 분리 매칭

### JD 키워드 DB (초안)

| 구분 | 키워드 |
|------|--------|
| 공통 | 4M, 개선활동, 불량, 품질, 공정, 표준화, 생산성, 원가, 현장, 실습 |
| 생기 | 설비, 자동화, 라인, 택타임, 시뮬레이션, 스마트팩토리, 제조원가, 치공구 |
| 품질 | FMEA, PPM, 8D, SPC, 6시그마, 공정능력, 검사, QC, 불량률, 개선 |
| 공정기술 | IE, UPH, 싸이클타임, 공수, 레이아웃, 표준작업, 공정설계, 동작분석 |

## 3. 채점 구조: 3레이어 모델

```
┌──────────────────────────────────────────────┐
│ Layer 1: 유사도 점수 (합격자 벡터군 대비)        │ ← 합격가능성 본체
│   "합격 자소서 분포상 상위 N%"                   │
├──────────────────────────────────────────────┤
│ Layer 2: 루브릭 가점 (있으면 +, 없어도 감점 X)   │ ← 보너스
│   수치 / 현장경험 / 자격증 / STAR / JD키워드      │
├──────────────────────────────────────────────┤
│ Layer 3: 신뢰도 플래그                          │ ← 멘토 유도 트리거
│   수치빈약 / 추상어과다 → "멘토 검증 권장"        │
└──────────────────────────────────────────────┘
```

| 레이어 | 역할 | 사용처 |
|--------|------|--------|
| Layer 1 (유사도) | 합격 가능성 판정 | 2축 모델의 Y축 점수 |
| Layer 2 (루브릭) | 강점 설명/피드백 | 사용자 피드백 코멘트 |
| Layer 3 (신뢰도) | 멘토 상담 유도 | 전환 트리거 |

> **합격 여부는 Layer1(유사도)이 결정하고, 왜 그런지는 Layer2(루브릭)로 설명한다.**
> → "블랙박스 AI"가 아니라 "근거 있는 진단"이 되는 핵심.

## 4. 문항별 채점 (문항 유형 5종)

문항마다 묻는 의도가 다르므로, 같은 항목이라도 유형별 가중치를 다르게 둔다.

### 문항 유형

| 유형 | 대표 질문 | 채점 핵심 |
|------|----------|----------|
| A. 지원동기 | "왜 현대차/이 직무인가" | 직무이해 + 회사특화 + 진정성 |
| B. 직무역량 | "직무 관련 경험/강점" | 경험구체성 + JD매칭 |
| C. 도전/성취 | "도전적으로 이룬 경험" | STAR + 정량성과 |
| D. 협업/갈등 | "팀에서 갈등 해결 경험" | 역할명확성 + 행동구체성 |
| E. 입사후포부 | "입사 후 목표/비전" | 직무이해 + 현실성 |

### 유형별 가중치 (생산/제조 버전, 각 문항 100점)

| 항목 | A.동기 | B.역량 | C.도전 | D.협업 | E.포부 |
|------|:---:|:---:|:---:|:---:|:---:|
| 경험 구체성 | 10 | **40** | **45** | 30 | 10 |
| 직무 연관성 | **35** | **35** | 20 | 15 | **35** |
| 논리 구조 | 20 | 10 | 15 | 20 | 20 |
| 차별성 | 10 | 10 | **20** | 25 | 15 |
| 회사/직무이해 | **25** | 5 | 0 | 10 | **20** |

> **굵은 칸** = 그 문항의 당락 결정 항목. 여기서 못 따면 그 문항은 끝.
> 생산/제조는 B·C에서 경험 구체성 배점이 특히 높다 (수치 없으면 신뢰도 급락).

### 문항 → 종합 점수 합산

```
문항별 원점수 (각 100점)
   ↓ 문항 중요도 가중
종합 자소서 점수 (100점 환산)
   ↓
2축 모델의 Y축으로 투입
```

문항 중요도 기본값 (직무별 조정 가능):

| 유형 | 가중 |
|------|:---:|
| B.직무역량 | 30% |
| A.지원동기 | 25% |
| C.도전경험 | 20% |
| D.협업 | 15% |
| E.포부 | 10% |

## 5. Layer 2 루브릭 세부 기준 (가점 방식)

> 모든 항목은 **가점**. 없다고 깎지 않는다. 멘토 검증으로 보낼 뿐.

### 5.1 수치 (정량성)
- 합격자 평균: **수치 3.1개 내외 포함** (예시값, 실제 DB로 갱신)
- 수치 많음 → 진단 신뢰도 HIGH, 점수 신뢰
- 수치 없음 → **감점 아님**. 진단 신뢰도 LOW → Layer3 플래그 발동
- 근거: "수치 없으면 무조건 불합격"이 아니라 "신뢰도가 떨어지고 그 확률이 높아짐" (문맥 의존)

### 5.2 현장경험 (인턴/현장실습)
- 있음 → 가점
- 없음 → **0 (감점 없음)**
- 근거: "없으면 불리하기보다 있으면 많이 도움된다"

### 5.3 자격증 (품질관리사, 6시그마 등)
- 자격증 + 다른 경험 동반 → 소폭 가점
- 자격증만 있고 다른 경험 부족 → **경고** ("자격증 외 경험 부족 신호")
- 근거: "있으면 땡큐지만, 그것만 어필하면 다른 경험 없다는 꼴"

### 5.4 STAR 구조
- 상황→행동→결과 완성도 (합격자 평균 72% 완성, 예시값)
- 완성 → 가점

### 5.5 JD 키워드 매칭
- 직무별 키워드 DB 대비 매칭 개수 → 가점

## 6. Layer 3 신뢰도 플래그 (멘토 유도)

다음 조건에서 "AI 진단 신뢰도 낮음 → 멘토 검증 권장" 표시:

| 플래그 | 조건 | 멘토 유도 메시지 |
|--------|------|-----------------|
| 수치 빈약 | 수치 < 합격자 평균의 50% | "구체적 수치가 부족해 AI 진단 신뢰도가 낮습니다" |
| 추상어 과다 | "열정/도전/성장/소통/협력/노력" 5개+ | "추상적 표현이 많아 정밀 진단이 어렵습니다" |
| 유사도 애매구간 | 합격자 분포 중위권(예: 30~60%) | "합격선 경계에 있어 현직자 확인이 필요합니다" |
| 자격증 과의존 | 자격증만 있고 경험 빈약 | "자격증 외 직무 경험 보강이 필요합니다" |

→ 모든 플래그는 **현직자 멘토 30분 상담**으로 자연 연결.

## 8. 2축 통합 모델 (스펙 × 자소서)

```
              자소서점수(Y)
                 100 |
                  80 |  [합격권]
                  60 |──────────── 합격선 (직무별 우하향 곡선)
                  40 |                      [애매구간 → 멘토 유도]
                  20 |                                  [탈락권]
                   0 +─────────────────────────────────→ 스펙점수(X)
                     0    20   40   60   80   100
```

- 스펙↑ → 자소서 요구수준↓ (우하향 곡선)
- 애매구간(곡선 근처) → 멘토 상담 유도
- 곡선 기울기는 직무별로 다름 (생산/제조는 스펙 민감도 낮은 편)
